  • Arduino Weather Monitor

    The Rochester weather can be very unpredictable, and to help me see what is going on I build a desktop weather monitor. The monitor consists of an Arduino with an 8x32 LED matrix. The Arduino is plugged into a raspberry pi, which runs a Java program to fetch weather data. The Java program uses the free API Open Weather Map(OWM). The program makes an HTTP request to the API which returns a JSON object with a bunch of weather data. The program takes out the temperature and an additional description such as sunny or cloudy. The program then sends the data the to the arduino over a serial connection

  • Mandelbrot Set

    You may not know the name, but you've probably seen many pictures of fractals online. They are the colorful images where you can infinitely zoom in and the never loose complexity. One particularly famous fractal is called the Mandelbrot set. The Mandelbrot set is a fractal based on the limit of the equation f(z) = z^2 + c, where c is a complex number. The function is recursive in nature because it starts out with z being 0, then in the next iteration of the equation c stays constant and z is the value result of the last equation. A number is said to be in the Mandelbrot set if the limit of that equation is less than 0.
    I created a program that creates visualizations of the set in python. First, I use the Python Imaging Library to create an image of 1024x1024. The user enters the bounds of the window as a command line argument, and the program goes pixel by pixel determining the complex number value of that pixel, and if the number is in the set. If it is in the set it is colored black and if not it is colored depending on how long it takes to escape the set.
    Here is a view of the generated Mandelbrot set. Code can be viewed on GitHub.

  • Place (Reddit) Clone

    Place was a social experiment created by reddit.com where users could place one pixel every 5 minutes on a shared canvas. A clone was created using Java and JavaFX. The concepts of Object Oriented Programming and Model View Controller were used to implement a server and client application.
    Sadly I cannot share the code for it on GitHub because it was my final project for CS for AP Students, but I can still discuss the project. The project was very enjoyable and taught me a lot about coding. It used many concepts such as Object Oriented Programming, Model View Controller, Networking, Threading, GUI, and input/output streams. Because I was able to work with one other person I was able to learn how to work on a project with another person.
    The main structure of the project consisted of two applications, a server, and an application, both were programmed in Java. The server was multithreaded so it could handle any number of users. It communicated with the application by creating a socket and sending or receiving messages using input or output streams.
    The application used JavaFX to display a simple grid of rectangles that can be colored by clicking on them. The user had to option of 16 colors to choose from. The core of the application revolved around the Model View Controller (MVC) principle. This principle made it much easier to code the application because the logic was completely separated from the UI. I could make changes to the UI without having to worry about affecting the logic. One of the things that surprised me was how easy it was creating different UIs using MVC. For example, the GUI was made first, but we also had to make a command line interface. Because of the separation of logic because of MVC, I was easily able to create a command line UI by just making a couple of calls to the application to get the data I needed.
    Beyond the base application, my partner and I created multiple bots. I created a bot that would take any image and place it on the board. I had the main requirement I had for the bot was to be able to preserve color as best as possible. Because there were only 16 color options for the board, but an image could have thousands of colors, I had to convert the colors of the image to one of the 16 colors. To do this I did a form of weighted average. I would compare the color of each pixel to each available color and take a weighted difference between the red, green, and blue component of each available color. The color with the lowest result would be the closest to the pixels color. This method worked very well. Most of the color of the images were maintained. I did notice the method did not work that well for human skin tones, but I was still happy with it.
    This was a large and challenging project, but it was very enjoyable. It allowed me to be creative and incorporated what we learned in class very well.
